## 附录：维护规则

### Agent 工作流

1. **开始任务前**：读取本文件确定当前重点和下一个可执行的 Task。
2. **执行任务时**：将对应 Task 状态改为 `in_progress`，同步更新本文件中该 Task 的状态列。
3. **完成任务后**：将 Task 状态改为 `done`，更新 Story 的 Task 进度（如 `2/4`）；若 Story 所有 Task 均 `done`，将 Story 标记为 `done`；若 Epic 所有 Story 均 `done`，将 Epic 标记为 `done` 并移入「已完成」区。
4. **遇到阻塞时**：将 Task 标记为 `blocked`，在「阻塞项与风险」中记录原因。
5. **状态同步**：本文件中的状态必须与 `stories/` 和 `tasks/` 下的源文件保持一致。

### 状态词汇

- `planned` — 已规划，未开始
- `in_progress` — 执行中
- `blocked` — 被阻塞
- `done` — 已完成
- `cancelled` — 已取消（不再实施）

### 变更规则

- 不复用已完成的 Epic / Story 条目承载新范围。
- 新增工作创建新的 ID，在源文档中说明关联关系。
- 本文件只做状态汇总，完整细节保留在 Epic / Story / Task 源文件中。

### 更新检查清单

每次更新本文件后确认：

- [ ] 「当前重点」与实际执行一致
- [ ] 每个 Task 的状态与源文件一致
- [ ] 每个 Story 的 Task 进度计数正确
- [ ] 阻塞项信息是最新的
- [ ] 已完成的条目已从活跃区移入「已完成」区
